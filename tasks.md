# 実施予定のタスク一覧
実施したタスクは tasks.md から削除した上でコミットしてください。
コミット単位はタスクごとにしてください。

## チケット一覧画面にソートを追加
id または updated でソート（昇順、降順）をできるようにしてください。
ソート指定した場合はクエリストリングにも反映してください。

## 履歴の閲覧・差分表示
チケット詳細画面から `ticket_histories` / `comment_histories` の履歴を閲覧できるようにしてください。
バージョン間の差分表示と、選択した版の内容に戻す機能を追加してください。
必要なAPI（例: `GET /api/tickets/{id}/histories`）も追加してください。

## NOT / OR 検索
タグ検索の `tags` パラメータを拡張してください。
`-status:CLOSE` のような先頭 `-` で除外（NOT）、`status:OPEN|status:WIP` のような `|` 区切りでOR条件を指定できるようにしてください。
チケット一覧画面のタグフィルタUIからも指定できるようにしてください。

## 保存済み検索（ビュー）
よく使う検索条件（`q` + `tags` の組み合わせ）に名前を付けて保存し、チケット一覧画面から切り替えられるようにしてください。
保存先は localStorage としてください。

## 数値タグ
日時タグ（グループ名末尾 `@`）と同様の仕組みで、グループ名末尾 `#` を数値タグとして扱ってください。
例えば `estimate#:3` のように数値を持ち、検索時に `estimate#:>=2` のように比較演算子で範囲指定できるようにしてください。

## チケット一覧画面でのタグによるソート
日時タグや数値タグでソートできるようにしてください。

## 画像の貼り付け添付
チケット・コメントの編集エリアに画像をペーストすると、画像が保存され markdown の画像リンクが挿入されるようにしてください。
画像の保存先はDBの想定です。配信APIも合わせて設計してください。

## エクスポート/インポート
チケット（コメント含む）を markdown または JSON でエクスポートできるようにしてください。
エクスポートしたデータをインポートする機能も追加してください。
