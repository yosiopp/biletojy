# 実施予定のタスク一覧
実施したタスクは tasks.md から削除した上でコミットしてください。
コミット単位はタスクごとにしてください。
関連の少ないタスクは、それぞれサブエージェント（Agentツール）に委譲してコンテキストを節約してください。
- 調査・分析タスクは並列実行して構いません。
- ファイルを変更するタスクを並列実行する場合は worktree で分離してください。

## チケット一覧での階層タグ順での並び替え

チケット一覧の「並び替え」プルダウンで、階層タグ（`a/b/c` 形式・グループ無し）のパス順にソートできるようにする。

- 現状のソートは `front/src/lib/sort.ts` のクライアントサイド実装（`SortSpec`、キーは `id` / `updated` / 日時・数値タググループ）。ここに階層タグのソートキーを追加する
- ソート値はチケットが持つ階層タグのパス文字列。`/` 区切りのセグメント単位で辞書順比較する（単純な文字列比較だと `a/b-x` と `a/b/c` の順が崩れるため）
- チケットが複数の階層タグを持つ場合は最小（先頭）のものをソート値とする
- 階層タグを持たないチケットは昇順・降順に関わらず末尾（既存の日時・数値タグソートと同じ挙動）
- ソートキーの選択肢: 階層タグはグループを持たないため、プルダウンには「階層タグ」のような固定の1項目を追加する（`sort=hierarchy` など。URLパラメータの表現は `parseSort` / `buildSort` に合わせる）

## 選択した階層タグを基にチケットをツリー表示するVIEW（表示モード）

チケット一覧に「リスト / ツリー」の表示モード切替を追加し、ツリーでは階層タグのパス構造でチケットを入れ子表示する。

- ここでの「VIEW」は表示モードのこと。既存の `ViewSelect`（保存済み検索条件のビュー）とは別物なので、UI上の呼称・実装で混同しないようにする
- 表示モードはURLパラメータで保持する（例: `view=tree`。検索条件 `q` / `tags` と共存させる）
- 表示対象の指定は `by` パラメータで行う（例: `view=tree&by=docs`。カンバンVIEWの基準グループ指定と共用する）
- ツリーの構造は絞り込み結果のチケットが持つ階層タグから構築する。中間ノードの列挙は `front/src/lib/tags.ts` の `hierarchyOptions` と同様のプレフィックス展開が使える
- ルート（例: `docs` だけ表示したい）を `by` で選択できるようにする。未選択時は全階層タグを対象とする
- 階層タグを持たないチケットは「未分類」ノードにまとめて表示する（隠さない）
- ノードの開閉ができること。キーボード操作（↑↓移動、←→で開閉など）で完結すること（プロジェクト要件）
- 複数の階層タグを持つチケットは各パスの下にそれぞれ表示して良い（重複表示を許容）
- 保存済みビュー（`lib/views.ts` の `SavedView`）に表示モード（`view`）・対象ルート（`by`）も含めて保存できるようにする（ノードの開閉状態は保存対象外）
- デザインは docs/design-system.md に従う。モバイル（sm未満）でのレイアウトも考慮する

## タググループを元にしたカンバンVIEW（表示モード）

タググループ（`status:` など排他的属性）を1つ選び、その値ごとの列にチケットカードを並べるカンバン表示モードを追加する。

- 上のツリーVIEWと同じ表示モード切替の仕組みに載せる。表示モードは `view=board`、基準タググループは `by` で指定する（例: `view=board&by=status`）
- 列はタグカタログのグループ値（`groupCatalog` で取得。カタログ順）+ 「なし」列（そのグループのタグを持たないチケット）
- 対象グループの選択はプルダウン。日時（`@`）・数値（`#`）グループは値が離散でないため対象外とする
- カードをドラッグ&ドロップで別の列に移すと、チケットのタグを付け替えて保存する（既存のチケット更新APIを利用。グループタグは排他なので旧値を外して新値を付ける）
- D&Dと等価の操作がキーボードでできること（例: カードにフォーカスして ←→ で列間移動）（プロジェクト要件）
- 各列内の並びは既存のソート指定（`sort` パラメータ）を流用する
- 検索条件（`q` / `tags`）による絞り込みはカンバンでも有効。列数が多い場合は横スクロール、モバイルでは列を縦積みにするなど docs/design-system.md に沿って調整する
- 保存済みビュー（`lib/views.ts` の `SavedView`）に表示モード・対象グループも含めて保存できるようにする
