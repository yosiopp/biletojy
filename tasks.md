# 実施予定のタスク一覧
実施したタスクは tasks.md から削除した上でコミットしてください。
コミット単位はタスクごとにしてください。
関連の少ないタスクは、それぞれサブエージェント（Agentツール）に委譲してコンテキストを節約してください。
- 調査・分析タスクは並列実行して構いません。
- ファイルを変更するタスクを並列実行する場合は worktree で分離してください。

## タググループを元にしたカンバンVIEW（表示モード）

タググループ（`status:` など排他的属性）を1つ選び、その値ごとの列にチケットカードを並べるカンバン表示モードを追加する。

- 上のツリーVIEWと同じ表示モード切替の仕組みに載せる。表示モードは `view=board`、基準タググループは `by` で指定する（例: `view=board&by=status`）
- 列はタグカタログのグループ値（`groupCatalog` で取得。カタログ順）+ 「なし」列（そのグループのタグを持たないチケット）
- 対象グループの選択はプルダウン。日時（`@`）・数値（`#`）グループは値が離散でないため対象外とする
- カードをドラッグ&ドロップで別の列に移すと、チケットのタグを付け替えて保存する（既存のチケット更新APIを利用。グループタグは排他なので旧値を外して新値を付ける）
- D&Dと等価の操作がキーボードでできること（例: カードにフォーカスして ←→ で列間移動）（プロジェクト要件）
- 各列内の並びは既存のソート指定（`sort` パラメータ）を流用する
- 検索条件（`q` / `tags`）による絞り込みはカンバンでも有効。列数が多い場合は横スクロール、モバイルでは列を縦積みにするなど docs/design-system.md に沿って調整する
- 保存済みビュー（`lib/views.ts` の `SavedView`）に表示モード・対象グループも含めて保存できるようにする
